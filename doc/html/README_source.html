9 INTRODUCTION
10 
11  Gesture is a touch input listener for linux systems. It detects configurable
12  gestures on the touch device and sends the associated keypresses. This
13  occurs at the system level using event streams, so it works anywhere, even
14  in a terminal with no display server. As a result, it is relatively
15  portable, as it works on any linux system with a touch device.
16 
17  Gesture is designed to enable power users to run a minimal interface on
18  their devices, either with no GUI, or with a mostly keyboard driven
19  GUI. This can be done all without a keyboard. However, it will also work
20  (albeit with little use) on a laptop touch pad.

30 USAGE
31 
32  ./gesture
33  <type configuration into stdin>
34 
35  Root may be required to access the system level event streams. By default,
36  a press that does not leave the center region indicates that the next
37  press should be ignored and act as a regular mouse action.
38 
39  Note: At the moment, usage inside an X server (I haven't tested any other
40  display servers yet) does not work because X steals all input events,
41  preventing Gesture from reading them. However, it will work in a terminal
42  (usually accessed with ctrl-alt-f#).
43 
44 CONFIGURATION
45 
46  Config data is passed to gesture through the standard input. A default
47  configuration file is included (see ./config), and it can be used through
48  redirection ('<'). You can use config files in this manner, or simply type
49  the settings in each time you run it.
50 
51  The first line is the path to the event stream to listen to. This usually
52  takes the form of "/dev/input/event#". You can determine which event stream
53  to use by checking each one with cat while touching the input device.
54 
55  The remaining lines each consist of two sequences separated by a space, but
56  with no spaces inside each one. The first is the sequence of regions in the
57  gesture. This will always start with 0, because each gesture must start at
58  the center. Each remaining number is the identifier for the region: 1 for up,
59  2 for right, 3 for down, and 4 for left. The second sequence is the list of
60  keypresses to make whenever the specified gesture is performed. Each
61  character may be any letter, number, or symbol that appears when a key is
62  pressed (such as a, b, and c, but not A, B, and C). Additionally, special
63  keys may be used enclosed in <>, such as <space>, <enter>, and <shift>.
